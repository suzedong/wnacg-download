# WNACG Downloader - 项目规则

## 项目概述

**项目名称**：WNACG Downloader

**产品形态**：CLI + Web 应用 + 桌面客户端三架构

**核心功能**：搜索、对比、下载汉化漫画

**目标用户**：需要批量下载汉化漫画的用户

**架构特点**：
- CLI：命令行工具，支持脚本化
- Web：浏览器访问，跨平台
- Tauri 2：桌面应用，原生体验（Rust 后端）
- 核心业务逻辑完全复用
- Web 和 Tauri 共享 UI 组件（复用率 > 95%）

## 功能规则

### 代理配置
- 默认使用配置文件中的代理设置
- 支持通过命令行参数覆盖代理设置
- 安装 Playwright 浏览器时使用代理加速下载

### 搜索规则
- 爬取所有页面，无页数限制
- **不启用缓存**：每次都从网站获取最新数据
- 只搜索汉化版漫画（可通过 --all 选项包含所有漫画）
- 爬取地址：`https://www.wnacg.com/search/index.php?q={keyword}&m=&syn=yes&f=_all&s=create_time_DESC&p={page}`
- 爬取流程：
  1. 使用关键字构建搜索 URL，初始页码为 1
  2. 访问第一页，提取总页数信息
  3. 解析第一页的漫画信息（进行去重）
  4. 并行打开所有剩余页面，提高爬取效率
  5. 解析所有页面的漫画信息
  6. 保存漫画信息到文件，方便后面对比和下载使用
- 浏览器模式：默认使用非无头模式，以便手动完成 Cloudflare 验证码
- **请求间隔**：默认 **1000ms**（平衡速度和礼貌爬取）
- 数据存储：搜索结果会被保存到 `cache/search_{keyword}.json` 文件中，后续命令会直接使用该文件，避免重复搜索
- 分类判断：通过元素的 `cate-*` 类名来判断漫画分类，例如 `cate-9` 对应 "單行本／漢化"，`cate-10` 对应 "雜誌&短篇／漢化" 等

**注意**：搜索功能不启用缓存机制，每次搜索都会从网站获取最新数据。保存的 JSON 文件供对比功能使用。

### 下载规则
- 自动对比本地已有漫画，避免重复下载
- 支持并发下载，默认并发数为 3
- 下载失败时会尝试重试
- 支持断点续传
- 下载完成后校验文件完整性

### AI 匹配规则
- 使用 AI 模型进行漫画名智能匹配，提高对比准确性
- 支持本地 AI 模型和远程 API 调用
- 内置增强的相似度计算算法，处理漫画名的变体
- **启用缓存机制**，提高匹配性能（仅 AI 匹配缓存）
- 匹配阈值可通过配置调整，默认为 0.8

## 配置管理

### 配置项清单
1. 默认存储路径
2. 代理地址（可选）
3. 最大爬取页数（默认 5）
4. **请求间隔**（毫秒，默认 1000）
5. 并发下载数（1-10，默认 3）
6. 只搜索汉化版漫画（布尔值）
7. 下载重试次数（默认 3）
8. 下载重试间隔（秒，默认 30）
9. AI 模型选择（本地/远程 API）
10. AI 模型 API 地址（如使用远程）
11. 匹配阈值（默认 0.8）

### 配置存储
- **使用 conf 库默认路径**（用户数据目录）
- 符合 Tauri 应用规范
- 用户数据和应用分离

## 核心设计理念

- **CLI、Web、Tauri 复用同一套业务逻辑**
- 核心模块：`src/core/`（scraper, downloader, scanner, comparer, ai）
- **Web 和 Tauri 共享 UI 组件**：`src/ui/`
- **适配器模式**：统一通信接口（HTTP vs Tauri Commands）
- CLI、Web、Tauri 只是交互方式不同
