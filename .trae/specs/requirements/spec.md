# WNACG Downloader - 需求规格说明书

## 1. 项目概述

WNACG Downloader 是一个支持 CLI、Web 应用和桌面客户端三模式的漫画下载工具，支持从 wnacg.com 网站搜索、对比和下载汉化漫画。

### 1.1 产品形态

**三架构设计**：
1. **CLI 工具**：命令行界面，支持脚本化、自动化操作
2. **Web 应用**：浏览器访问，跨平台，无需安装
3. **桌面客户端**：Electron 打包，提供原生体验

**架构关系**：
- CLI：独立运行，核心业务模块
- Web：Express 服务器 + Vue 前端，核心业务模块复用
- Electron：桌面封装，核心业务模块复用

**未来规划**：CLI 将包装成 Skill，支持更广泛的集成场景

### 1.2 技术栈

#### 共享核心模块
- **核心功能**：Playwright（爬虫）+ Axios（下载）
- **AI 匹配**：支持本地模型和远程 API 切换
- **配置管理**：conf 库
- **日志系统**：winston

#### CLI 专用
- **命令行框架**：Commander.js
- **交互式 TUI**：Inquirer.js + Chalk + Figures
- **终端 UI**：自定义 TUI 界面

#### Web 专用
- **后端框架**：Express.js
- **前端框架**：Vue 3 + TypeScript + Vite
- **API 通信**：HTTP RESTful API
- **部署方式**：Node.js 直接运行 / Docker

#### 客户端专用
- **前端框架**：Vue 3 + TypeScript + Vite（与 Web 共享）
- **桌面框架**：Electron
- **IPC 通信**：Electron IPC
- **组件系统**：可复用 Vue 组件（与 Web 共享）

### 1.3 UI 设计策略

**Web 和 Electron 共享同一套 UI 组件**：

**设计原则**：
- ✅ **组件完全复用**：Web 和 Electron 使用相同的 Vue 组件
- ✅ **样式完全一致**：共享同一套 CSS 样式和主题
- ✅ **交互完全相同**：用户操作流程一致
- ✅ **代码复用率 > 95%**：最大化代码复用

**实现方式**：
```
src/ui/                    # 共享 UI 组件
├── components/            # 可复用组件（卡片、按钮等）
├── views/                 # 页面组件（搜索、对比、下载等）
├── styles/                # 共享样式
└── composables/           # 共享逻辑（Composition API）

src/web/                   # Web 专用
└── main.ts                # Web 入口（调用共享 UI）

src/electron/              # Electron 专用
├── main.ts                # Electron 主进程
├── preload.ts             # 预加载脚本
└── renderer/
    └── main.ts            # 渲染进程入口（调用共享 UI）
```

**差异处理**：
- **通信方式**：
  - Web：通过 HTTP API 与后端通信
  - Electron：通过 IPC 与主进程通信
  - 抽象统一的接口层，UI 组件无感知

- **打包方式**：
  - Web：Vite 打包成静态文件
  - Electron：Vite 打包 + Electron 打包

**优势**：
- ✅ 开发效率高（一套代码，多处运行）
- ✅ 维护成本低（修改一次，多端生效）
- ✅ 用户体验一致（Web 和 Electron 界面完全相同）
- ✅ 测试成本低（UI 测试用例可复用）

## 2. 核心功能需求

### 2.1 功能 1：搜索漫画

#### 2.1.1 功能描述
从 wnacg.com 网站搜索汉化漫画，支持关键字搜索和并行爬取。

#### 2.1.2 功能流程
1. 用户输入作者/关键字
2. 构建搜索 URL，访问第一页
3. 提取总页数信息
4. 并行爬取所有页面
5. 解析漫画信息（去重）
6. 显示搜索结果（完整列表，可滚动）
7. 用户预览搜索结果（确认搜索是否正确）
8. 保存结果为 JSON 文件（供对比使用）

#### 2.1.3 功能规则
- ✅ 不启用缓存：每次都从网站获取最新数据
- ✅ 只搜索汉化版漫画（可通过配置关闭）
- ✅ 爬取所有页面，无页数限制
- ✅ 非无头模式浏览器，支持手动完成 Cloudflare 验证码
- ✅ 通过 `cate-*` 类名判断漫画分类
- ✅ **请求间隔：默认 1000ms**（平衡速度和礼貌爬取）
- ✅ 同关键字搜索结果只保留一个文件（新搜索自动覆盖）
- ✅ 搜索完成后显示完整结果列表（可滚动）
- ✅ 显示信息：漫画标题、作者、分类、页数、封面图（UI 模式）
- ✅ 重新搜索时，如关键字相同，需用户确认覆盖

#### 2.1.4 数据存储
```
cache/search_{keyword}.json
```

#### 2.1.5 数据结构

- ✅ 同关键字只保留一个文件：`search_{keyword}.json`
- ✅ 新搜索自动覆盖旧文件

- 显示搜索结果列表页面

- 按时间倒序显示（最新在上）
- 支持关键字过滤（过滤显示结果，不删除文件）

**UI 实现**：
- 对比页面显示搜索结果列表

- 显示信息：关键字、保存时间、漫画数量、文件大小
- 按时间倒序显示（最新在上）
- 支持关键字过滤（过滤显示结果，不删除文件）
- 点击选中后开始对比

#### 2.1.7 搜索结果预览功能

**功能描述**：

### 2.2 功能 2：对比本地漫画

#### 2.2.1 功能描述
对比网站搜索结果、本地已有漫画和第三方网站数据，智能匹配并生成下载清单。

#### 2.2.2 功能流程
1. 用户选择搜索结果（通过关键字或文件列表）
2. 从缓存的 JSON 文件加载搜索结果
3. 选择对比基准（本地文件夹 + 第三方网站）
4. 扫描本地漫画文件夹（仅扫描一层）
5. 加载第三方网站数据（待设计）
6. 使用 AI 模型进行三方智能匹配
7. 以第三方网站名称为准进行校准
8. 显示对比结果（匹配详情）
9. 人工确认最终需要下载的漫画
10. 保存对比结果 JSON
11. 生成下载清单 JSON
12. 跳转到下载页面

#### 2.2.3 功能规则
- ✅ AI 模型支持本地和远程 API 切换（通过配置）
- ✅ 内置增强相似度算法，处理变体
- ✅ 匹配阈值默认 0.8（可配置）
- ✅ 启用缓存机制提高匹配性能
- ✅ 本地文件夹只扫描一层（不递归）
- ✅ 第三方网站数据作为权威数据源（名称校准基准）
- ✅ 支持重新对比功能
- ✅ 支持保存对比结果
- ✅ 对比结果自动保存（带时间戳）

#### 2.2.3A CLI 对比命令

**命令格式**：
```bash
wnacg-dl compare [keyword] [options]

参数:
  keyword              搜索关键字（可选，不传则交互选择）

选项:
  -l, --local <path>   本地漫画路径
  -t, --third-party    启用第三方对比（待实现）
```

**交互选择示例**：
```bash
$ wnacg-dl compare

请选择搜索结果（输入关键字）:
  1. TYPE90 (2026-04-11 10:30, 42 部)
  2. 作者名 A (2026-04-10 15:20, 28 部)
  3. 关键字 B (2026-04-10 12:15, 15 部)

输入关键字：TYPE90
```

**错误提示**：
```bash
$ wnacg-dl compare 不存在的名字

❌ 未找到搜索结果：不存在的名字
💡 提示：使用 wnacg-dl search --list 查看所有搜索结果
```

**对比结果保存**：
```
cache/compare_{keyword}_{timestamp}.json

示例：
- compare_TYPE90_20260411_103000.json
- compare_作者名 A_20260410_152000.json
```

#### 2.2.4 对比基准选择
- 本地文件夹（可选）
- 第三方网站（可选，权威数据源）
- 支持同时选择本地 + 第三方

## 3. 配置管理需求

### 3.1 配置项清单
1. **默认存储路径**：漫画下载存储路径
2. **代理地址**：网络代理地址（可选）
3. **最大爬取页数**：搜索时最大爬取页数（默认 5）
4. **请求间隔**：请求间隔时间（毫秒，**默认 1000**）
5. **并发下载数**：并发下载数量（1-10，默认 3）
6. **只搜索汉化版漫画**：布尔值（默认 true）
7. **下载重试次数**：下载失败重试次数（默认 3）
8. **下载重试间隔**：重试间隔时间（秒，默认 30）
9. **AI 模型选择**：本地模型/远程 API
10. **AI 模型 API 地址**：远程 API 地址（如使用远程）
11. **匹配阈值**：AI 匹配相似度阈值（默认 0.8）

### 3.2 配置规则
- ✅ 配置文件存储在**conf 库默认路径**（用户数据目录）
- ✅ 符合 Electron 应用规范
- ✅ 用户数据和应用分离
- ❌ 不需要配置导入/导出功能
- ✅ 支持缓存文件管理（删除、清空）

### 3.3 配置验证规则

**请求间隔**：
- 配置项：`requestDelay`
- 有效范围：0-10000ms
- 默认值：1000ms
- 说明：0 表示无间隔，10000 表示 10 秒
- CLI 参数：`--delay <ms>`
- UI 配置：配置页面 > 搜索设置 > 请求间隔

**最大爬取页数**：
- 配置项：`maxPages`
- 有效范围：1-100
- 默认值：5

**并发下载数**：
- 配置项：`concurrentDownloads`
- 有效范围：1-10
- 默认值：3

**下载重试次数**：
- 配置项：`retryTimes`
- 有效范围：0-10
- 默认值：3

**下载重试间隔**：
- 配置项：`retryInterval`
- 有效范围：10-300 秒
- 默认值：30 秒

**匹配阈值**：
- 配置项：`matchThreshold`
- 有效范围：0.5-1.0
- 默认值：0.8

### 3.4 缓存管理功能

**功能描述**：
管理搜索结果和对比结果生成的缓存文件，支持查看、删除和清空操作。

**缓存文件类型**：
- 搜索结果缓存：`cache/search_*.json`
- 对比结果缓存：`cache/compare_*.json`

**CLI 命令**：
```bash
# 列出所有缓存
wnacg-dl cache list

# 输出示例
缓存文件列表:

搜索结果:
  1. TYPE90 (2026-04-11 10:30, 42 部，2.3 MB)
  2. 作者名 A (2026-04-10 15:20, 28 部，1.5 MB)
  3. 关键字 B (2026-04-10 12:15, 15 部，0.8 MB)

对比结果:
  1. compare_TYPE90_20260411_103000 (2026-04-11 10:35, 7 部待下载，1.2 MB)
  2. compare_作者名 A_20260410_152000 (2026-04-10 15:25, 3 部待下载，0.5 MB)

总占用：6.3 MB
```

```bash
# 删除指定缓存（支持多个）
wnacg-dl cache delete search_TYPE90
wnacg-dl cache delete search_TYPE90 search_作者名 A
wnacg-dl cache delete compare_TYPE90_20260411_103000

# 清空所有缓存
wnacg-dl cache clear
```

**删除确认**：
- 删除前需要用户确认
- 显示要删除的文件列表
- 显示将释放的空间
- 示例：
  ```
  ⚠️ 确认删除
  
  确定要删除选中的 3 个缓存文件吗？
  
  将删除：
  • search_TYPE90.json (2.3 MB)
  • search_作者名 A.json (1.5 MB)
  • compare_TYPE90_....json (1.2 MB)
  
  总释放空间：5.0 MB
  
  💡 提示：删除缓存不会影响已下载的漫画
  
       [取消]        [确认删除]
  ```

**UI 实现**：
- 配置页面新增"缓存管理"标签
- 显示搜索结果缓存和对比结果缓存
- 显示信息：文件名、保存时间、漫画数量、文件大小
- 支持多选删除
- 显示总占用空间
- 操作按钮：
  - [全选]
  - [取消选择]
  - [删除选中]
  - [清空全部]
- 删除前弹窗确认（显示文件列表和释放空间）
- 提示信息：删除缓存不会影响已下载的漫画

### 4.2 CLI 界面需求

#### 4.2.1 命令行接口

**命令结构**：
```bash
wnacg-dl <command> [options]

commands:
  search     搜索漫画
  compare    对比本地漫画
  download   下载漫画
  config     配置设置
```

**搜索命令**：
```bash
wnacg-dl search <keyword> [options]
  -p, --pages <number>     最大爬取页数
  -o, --output <file>      输出 JSON 文件路径
  --all                    搜索所有版本（包括非汉化）
```

**对比命令**：
```bash
wnacg-dl compare <keyword> [options]
  -s, --storage <path>     本地存储路径
  -o, --output <file>      输出对比结果
  --third-party <source>   第三方数据源
```

**下载命令**：
```bash
wnacg-dl download <keyword|file> [options]
  -s, --storage <path>     存储路径
  -c, --concurrent <num>   并发下载数
  -y, --yes                自动确认
```

**配置命令**：
```bash
wnacg-dl config [options]
  --list                   列出所有配置
  --set <key>=<value>      设置配置项
  --get <key>              获取配置值
```
