# WNACG Downloader - 开发计划

## 项目状态

**当前版本**: v4.0（纯桌面端）
**技术栈**: Rust + Tauri 2 + Vue 3 + TypeScript
**整体完成度**: 约 85-90%

---

## 现状概览

### 已完成的核心功能

#### 后端（Rust）- 100%
- ✅ 配置管理系统（读写 JSON、默认配置、系统路径）
- ✅ 事件系统（5 种事件类型、进度推送）
- ✅ 错误处理（完整的错误类型覆盖）
- ✅ Rust 后端核心模块（downloader, scanner, comparer, ai）
- ✅ 搜索模块（Node.js + Playwright 脚本）
- ✅ scraper 模块（保留代码，当前未使用）
- ✅ Tauri Commands（search、download、compare、config、open_folder 全部实现）
- ✅ AI 匹配两阶段流程：本地精确/模糊匹配优先 + AI 仅兜底未匹配漫画（解决 81 部漫画一次性 AI 调用慢的问题）
- ✅ AI API 未配置时，未匹配漫画自动标记为 `need_download`
- ✅ 三级下载回退策略：Playwright Worker API → Server 2 直链 → 拼接直链兜底
- ✅ 系统托盘（显示窗口、退出应用、点击图标恢复窗口）
- ✅ 窗口控制（最小化、最大化、关闭隐藏）

#### 前端（Vue 3）- 80%
- ✅ 应用框架（Vue 3 + TypeScript + Vite）
- ✅ 主布局（App.vue、侧边栏导航）
- ✅ 主题系统（亮色/暗色模式、CSS 变量）
- ✅ 侧边栏（4 个导航项、主题切换按钮）
- ✅ Composables（useSearch、useDownload、useCompare、useConfig、useDownloadQueue）
- ✅ 类型系统（types/index.ts 统一定义、前后端一致）
- ✅ 配置页面（完整表单、所有配置项）
- ✅ 搜索页面（搜索结果展示、历史记录、队列联动、预览跳转）
- ✅ 对比页面（统计卡片、对比结果展示、需下载/已拥有分类）
- ✅ 下载页面（下载队列、进度显示、控制按钮）

## 关键问题与风险

### 已知问题
1. **暗色模式跟随系统**：当前仅支持手动切换

### 技术风险
1. **WNACG 网站结构变化**：HTML 解析可能失效
2. **AI API 成本控制**：已优化为「本地优先 + AI 兜底」，大部分情况零 API 调用
3. **并发下载稳定性**：大量任务时可能遇到资源竞争
4. **跨平台兼容性**：Windows 和 macOS 的托盘、通知差异

---

## 开发建议

1. **严格按照 Phase 顺序执行**，不要跳过
2. **Phase 1 是核心**，完成页面细节后功能基本闭环
3. **每个任务完成后立即验证**，不要堆积
4. **遇到问题先查文档**，不要盲目尝试
5. **保持代码整洁**，及时提交
6. **中文注释和日志**，保持一致性
