# WNACG Downloader - 界面设计文档

## 概述

本文档用图形化方式展示 WNACG Downloader 的界面设计结构，包括：
- **Web 应用**（Express + Vue 3）
- **桌面客户端**（Tauri 2 + Vue 3）
- **CLI 工具**（命令行界面 + 交互式 TUI）

**设计原则**：
- ✅ **Web 和 Tauri 共享同一套 UI 组件**（设计目标）
- ✅ **适配器模式统一通信接口**（设计目标）
- ✅ **代码复用率 > 95%**（设计目标）
- ✅ **轻量级**（无 UI 库依赖，所有组件自研）
- ✅ **紫色主题**（#667eea → #764ba2）
- ✅ **桌面优先**（基础响应式 ≥ 768px）

### 1.2 适配器模式设计（Web 和 Tauri 共享）

**Web 和 Tauri 的通信方式不同**：
- Web：通过 HTTP API 与 Express 服务器通信
- Tauri：通过 Tauri Commands 与 Rust 后端通信（Rust 调用 Node.js 子进程）

**解决方案**：使用适配器模式抽象统一接口

```
┌─────────────────────────────────────────────────────────────────┐
│                    UI 组件                                      │
│              (SearchView, CompareView, etc.)                    │
└────────────────────────┬────────────────────────────────────────┘
                         │ 调用
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│                   适配器层 ⭐                                    │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │  ISearchClient (接口)                                   │  │
│  │  + search(keyword, options): Promise<Comic[]>           │  │
│  │  + download(comics, options): Promise<DownloadResult>   │  │
│  │  + compare(searchFile, options): Promise<CompareResult> │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                                 │
│  ┌─────────────────┐           ┌─────────────────┐            │
│  │  ApiClient      │           │  TauriClient    │            │
│  │  (Web 专用)      │           │ (Tauri 专用)     │            │
│  │                 │           │                 │            │
│  │  fetch('/api')  │           │ invoke()        │            │
│  │                 │           │                 │            │
│  └─────────────────┘           └─────────────────┘            │
└─────────────────────────────────────────────────────────────────┘
```

**代码示例**：

```typescript
// 1. 定义接口
// src/ui/adapters/types.ts
export interface ISearchClient {
  search(keyword: string, options: SearchOptions): Promise<Comic[]>;
}

// 2. Web 实现
// src/ui/adapters/api-client.ts
export class ApiClient implements ISearchClient {
  async search(keyword: string, options: SearchOptions): Promise<Comic[]> {
    const response = await fetch('/api/search', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ keyword, ...options })
    });
    const data = await response.json();
    return data.comics;
  }
}

// 3. Tauri 实现（未来）
// src/ui/adapters/tauri-client.ts
import { invoke } from '@tauri-apps/api/core';

export class TauriClient implements ISearchClient {
  async search(keyword: string, options: SearchOptions): Promise<Comic[]> {
    return invoke('search_comics', { keyword, options });
  }
}

// 4. UI 组件使用（无感知）
// src/ui/views/SearchView.vue
<script setup lang="ts">
import { createClient } from '../adapters';

const client = createClient(); // 根据环境自动创建

const performSearch = async () => {
  const results = await client.search(keyword.value, options.value);
  // ...
};
</script>
```

**优势**：
- ✅ UI 组件完全无感知通信方式
- ✅ Web 和 Tauri 代码复用率 > 95%
- ✅ 易于测试（可以 Mock 客户端）
- ✅ 易于扩展（新增客户端实现即可）

### 2.5 错误提示策略（场景化）

**操作前错误**（模态框，需要用户确认）：
- 网络错误
- Cloudflare 验证失败
- 配置文件错误
- 覆盖确认

**操作中错误**（Toast 通知，自动消失）：
- 部分下载失败
- 重试提示
- 进度更新

**操作后错误**（页面显示）：
- 对比结果中的匹配失败
- 下载完成后的失败列表

### 2.6 批量操作确认策略

**不需要确认的操作**：
- ✅ 添加到下载队列（可逆操作）
- ✅ 批量下载 ≤ 10 个漫画

**需要确认的操作**：
- ⚠️ 删除操作（不可逆）

interface DownloadState {
  queue: DownloadTask[]          // 下载队列
  isDownloading: boolean         // 是否正在下载
  progress: DownloadProgress     // 下载进度
  result: DownloadResult | null  // 下载结果
}

interface DownloadTask {
  aid: string                    // 漫画 ID
  title: string                  // 标题
  url: string                    // 下载 URL
  coverUrl: string               // 封面图 URL
  pages: number                  // 页数
  status: 'pending' | 'downloading' | 'completed' | 'error'
  progress: number               // 进度 0-100
  speed?: string                 // 下载速度
  eta?: string                   // 剩余时间
  error?: string                 // 错误信息
}

interface DownloadProgress {
  total: number                  // 总数
  current: number                // 当前
  percentage: number             // 百分比
  speed: string                  // 速度
  eta: string                    // 剩余时间
}

interface DownloadResult {
  success: number                // 成功数量
  failed: number                 // 失败数量
  successList: string[]          // 成功列表
  failedList: FailedComic[]      // 失败列表
}

interface FailedComic {
  title: string
  reason: string
}
